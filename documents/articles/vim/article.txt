1. Запуск редактора.

1.1. Запуск Vim в режиме вставки.

По умолчанию Vim запускается в так называемом нормальном режиме, в которром
сосредоточено всё управление. Мы же с помощью маппинга перенесем нужное
нам управление в режим вставки, как это сделано в большинстве редакторов.
Поэтому и запускать Vim нам было бы удобнее сразу в режиме вставки. Чтобы 
это сделать, открываем для редактирования конфиг для bash ~/.bashrc и 
вставляем в него строчки:

    # запуск vim в режиме вставки
    alias vim="vim -c 'startinsert'"

Первая строка - комментарий, вторая говорит системе: если команда - 'vim',
то запускать то, что справа от знака равенства, т. е. Vim в режиме вставки.
Чтобы два раза не открывать конфиг, добавим в него еще строку

    export PATH="$PATH:~/.local/bin"

Это путь для запуска скрипта, создающего проект для редактирования в Vim 
(об этом речь далее).
Чтобы изменения в ~/.bashrc вступили в силу, перезагружаемся или 
запускаем команду

    $ source ~/.bashrc

2. Настройки для удобной работы. Сопоставления.
Как уже говорилось, ~/.vimrc - это конфиг для Vim; все настройки редактора
описываются в этом файле. Последняя версия моего конфига находится на GitHub:
https://github.com/shodanovich/vimrc/blob/master/.vimrc
Установочные команды всем известны, везде описаны и зависят в основном от 
личных предпочтений. Мои установки собраны из разных источников; все их 
описывать не буду. Опишу только самые необходимые.

Строка 8.
filetype plugin indent on

Эта опция необходима для определения типа файла, автоотступов, а также для 
запуска скриптов и плагинов.

Строка 10.
syntax on

Боевая раскраска.

Теперь сопоставления. Написаны они так, чтобы 
1) командовать редактором в основном из режима вставки и 
2) не отходить далеко от оригинальных команд Vim. 
Например, в нормальном режиме сочетание dd удаляет текущую строку. У меня в
режиме вставки сочетание Ctrl-d делает то же самое:

" удалить строку из режима вставки
inoremap <C-d> <Esc>ddi

Русшифровываем слово "inoremap" справа налево: "map" - сопоставление, "nore" - 
без рекурсии (везде это вставляю, чтобы Vim не запутался), "i" - режим вставки.
Далее, <C-d> - это одновременное нажатие Ctrl+d. Этому нажатию сопоставляется
последовательность команд <Esc> - переход в нормальный режим; dd - удаление
текущей строки; i - переход в режим вставки.

Продолжаем разбирать маппинг. Выделим фрагмент и проделаем с ним различные 
манипуляции:

" Ctrl-v - режим визуальный посимвольный
inoremap <C-v> <Esc>v
" удаление того, что выделено, с переходом в режим вставки
" (далее по Ctrl-p можем вставить фрагмент в другом месте; получится перенос)
vnoremap <Del> <Del>i
" копирование выделенного фрагмента с переходом в режим вставки
" (далее по Ctrl-p можем вставить фрагмент)
vnoremap y yi
" перевести выделенный фрагмент в верхний регистр
vnoremap U Ui
" перевести выделенный фрагмент в нижний регистр
vnoremap u ui

vnoremap - это сопоставление в визуальном режиме; есть ещё nnoremap - 
сопоставление в нормальном режиме, и мы его также будем использовать.

3.1. Другой подход.

Основная идея: в папке проекта создаем текстовый 
файл project, в котором находятся имена файлов проекта и далее работаем уже
с этим файлом. Было бы хорошо, чтобы при его открытии автоматически 
открылись бы файлы, перечисленные в нем. И чтобы работал REPL.
Для достижения этих целей надо сделать три вещи: создать файл 
project с именами файлов проекта, правильно открыть
его и настроить Vim для запуска нашего проекта.
Приступим.

имена файлов роекта, каждый с новой строки. Но, если файлов в проекте 
много, это становится утомительным. Для этой цели был написан простой

4. Примеры.

Проект на Ruby.

Средствами Ruby устанавливаем гем Bundle, формирующий структуру каталогов и 
файлов проекта. Переходим в папку с проектами (у меня ~/projects):

 $ cd ~/projects

 и запускаем гем:

 $ bundle gem project1

project1 - это название моего проекта. 
В каталоге ~/projects/project1 будет сформирована структура проекта. Там 
получается несколько разных папок, но нас будет интересовать папка bin (в ней
будут располагаться скрипты) и lib с будущими исходниками.
